# Next Market AI & Advanced Features Guide (AI機能・特殊システム詳細解説ガイド)

本リポジトリで新規開発および統合された、ハッカソンの評価基準を最高レベルに引き上げるための高度なAI機能と、独自のアルゴリズムシステムについて解説します。

---

## 1. 🤖 AI Agent Price Negotiation Marketplace（エージェント・フリマ）
「値下げ交渉、AIに丸投げしませんか？」をテーマに、購入者・出品者の双方が代理AIエージェントに値下げ交渉を委託できる自律交渉システム。

### 💡 技術的な見どころ
*   **自律型マルチエージェント対話**:
    `POST /api/items/{id}/negotiate` エンドポイントが呼び出されると、バックエンドが購入者AIエージェントと出品者AIエージェントをシミュレートし、最大6ターンの「自律対話ループ」を実行します。
*   **キャラクター・ロールプレイ (Prompt Engineering)**:
    出品者は交渉AIの性格（コテコテの大阪商人 / 冷静エリート / 元気なアニメキャラ / 標準）を設定可能。システムプロンプトに性格に応じたセリフ生成規則を完全に統合。
*   **トランザクション安全自動決済**:
    交渉が合意（Accept）に達した場合、その場でデータベーストランザクション（`SELECT FOR UPDATE` によるロック）を開始し、合意した値下げ価格で `purchases` レコードを作成。商品のステータスを `sold`（売却済み）に更新します。

## 2. 🔄 Barter Loop（AIマルチホップ物々交換プラットフォーム）
お金を使わない物々交換。3人以上のユーザー間で発生する需要の循環（A ➔ B ➔ C ➔ A）を自動検出し、わらしべ長者ネットワークを構築します。

### 💡 技術的な見どころ
*   **バックグラウンド・バッチマッチング (Goroutine Daemon)**:
    30分に1回バックグラウンドで自動起動する非同期デーモン `runBarterMatcher()` を搭載。待機ユーザー数が2人以上の場合のみ走り、OpenAI APIの無駄な消費を自動で防ぐ最適化設計。
*   **有向グラフの閉路検出アルゴリズム (DFS)**:
    出品アイテムのカテゴリーと、ユーザーが希望する「欲しいジャンル」の依存関係を「有向グラフ」としてモデル化。深さ優先探索（DFS）により、2者間・3者間の循環ループを瞬時に検出。
*   **AI適正価値査定 ＆ 差額調停自動決済**:
    OpenAI（`gpt-4o-mini`）を介して、ループ内の各商品の推定取引価値（市場価格や状態）を算出。全員の純利益が完全に「0」になり、プール内の清算差額の総和が「ちょうど0円」になるように差額調整金（受け取るか支払うか）を自動算出します。

## 3. 📸 AI使用風景生成 (Scene Generation)
商品画像とユーザーのプロフィール写真をAI（OpenAI Image Edit / DALL-E）を用いて自然にマージし、商品を使用している魅力的な合成画像を生成。

### 💡 技術的な見どころ
*   **マルチパート Mime-Type の解決**:
    Go標準の `CreateFormFile` がヘッダーを `application/octet-stream` に固定してOpenAI APIが弾かれるバグを解消。`textproto.MIMEHeader` を手動構築し、各画像アセットの元のMime-Type（`image/jpeg`, `image/png`）を動的にインジェクションして、OpenAI APIが要求する正しいマルチパートを生成します。
*   **データベース永続化**:
    `item_scene_generations` テーブルを介して、各ユーザーごとに生成された使用風景画像のパスを永続化し、商品詳細ページ上に美しく再読み込み。

## 4. 📊 ダブル・ダッシュボード (データ分析)
一般ユーザー向けの「マイ・ダッシュボード」と、管理者向けの「システム管理画面」の2重ダッシュボード。

### 💡 技術的な見どころ
*   **個人分析 (マイ・ダッシュボード)**:
    全ユーザーが、獲得売上総額、販売成立件数、出品中の商品、獲得いいね総数の4つのKPIおよび「カテゴリー別出品比率」「日別売上推移トレンド（直近30日）」をグラフ（バニラHTML/CSSプログレスバー）で可視化。
*   **システム分析 (管理者ダッシュボード)**:
    システム全体のKPI、およびOpenAIがテキスト・画像解析から不適切物や危険レベルを判定した「AI出品審査・モデレーション履歴ログ」のリアルタイム監視、ユーザー一覧・ロール変更（一般 ⇄ 管理者）を実現。
*   **カバリングインデックスによる高速クエリ化**:
    MySQL上に `idx_purchases_seller_created_at_price (seller_id, created_at, price)` などの複合インデックスを冪等に追加。集計クエリをディスクにアクセスせずメモリ上だけで解決。
