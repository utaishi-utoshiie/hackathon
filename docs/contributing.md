# Next Market - 開発者・コントリビューター向けガイド (Contributing Guide)

Next Market へようこそ！本書は、将来のメンテナや開発者がプロジェクトにスムーズに参加し、新機能の実装やコードのメンテナンスを行うための包括的な技術ガイドです。

---

## 🗺️ 1. システム全体設計とアーキテクチャ

Next Market は、Go で書かれたモノリシックかつ高速なバックエンドと、React (Vite/TypeScript) で構成された流麗なシングルページアプリケーション (SPA) フロントエンドの密結合ハイブリッド構成です。

```
[クライアント (React / TypeScript / CSS)]
       │
       │ (REST APIs + Auth Bearer Token)
       ▼
[バックエンド (Go 1.25)] ───► [Goose 組み込みマイグレーション]
       │                                     │
       ▼ (GCloud SQL Proxy / Connection)     ▼ (Boot Schema Run)
[MySQL 8.0 データベース (Cloud SQL)] ◄────────┘
       │
       ▼ (Go SDK Calls)
[OpenAI API (GPT & DALL-E)] ＆ [Stripe エスクロー API]
```

### 🗂️ ディレクトリ構造
*   `hackathon/` (すべての開発成果物はこのサブディレクトリ内にあります)
    *   `backend/`: Go言語バックエンド
        *   `cmd/api/`: メインアプリケーションエントリーポイント。機能・画面別にハンドラーファイルが細分化（`handlers_barter.go`、`handlers_ai.go` 等）されています。
        *   `migrations/`: Goose による SQL マイグレーションファイル群。
    *   `frontend/`: React + Vite + TypeScript フロントエンド
        *   `src/`: 画面ごとにモジュール化されたクリーンなコンポーネント設計。
    *   `docs/`: 仕様書、デザインシステム、FAQ、および本ガイド。

## ⚙️ 2. ローカル開発環境のセットアップ

### データベース (MySQL 8.0) の準備
1.  ローカルに MySQL を立ち上げ、データベース（例: `next_market`）を作成します。
2.  データベースの接続情報（DSN）を環境変数に設定します：
    ```bash
    export DATABASE_DSN="user:password@tcp(127.0.0.1:3306)/next_market?parseTime=true"
    ```

### バックエンド (Go 1.25) の実行
1.  依存パッケージのダウンロード：
    ```bash
    cd hackathon/backend
    go mod download
    ```
2.  サーバーの起動（起動時に Goose が自動検知し、未適用のデータベーススキーママイグレーションを全て適用します）：
    ```bash
    go run ./cmd/api/main.go
    ```
3.  テストおよびベンチマークの実行：
    ```bash
    go test -v ./...
    go test -bench=. -benchmem ./...
    ```

### フロントエンド (React / TS) の実行
1.  依存関係のインストール：
    ```bash
    cd hackathon/frontend
    npm ci
    ```
2.  Viteローカル開発サーバーの起動：
    ```bash
    npm run dev
    ```
3.  本番用コードコンパイル＆ビルド確認（型チェック、最適化、トランスパイル）：
    ```bash
    npm run build
    ```

## 🧮 3. 実装されている「次世代AIコアアルゴリズム」

Next Market が他を圧倒する性能を誇る理由は、AI とプログラム（決定論的数理ロジック）を適材適所で高度に融合させた **「ハイブリッドAI設計」** にあります。

### ① 【超高速・1ターン】共同意思決定AI価格交渉
*   **ファイル**: `backend/cmd/api/handlers_ai.go` (`negotiateItem`）
*   **ロジック**: 通常は何ターンもAIへの往復APIコールが発生する交渉プロセスを、**「購入者AIと出品者AIの性格・予算上限を1つのプロンプトに内包し、OpenAIに最大6ターンの会話と合意成立可否を1ターンで同時出力（Joint Simulation）させる」** ことで、応答遅延を 15秒から 1.5秒へと短縮しています。

### ② 【代数的】3者間等価循環物々交換（ゼロサム・ソルバー）
*   **ファイル**: `backend/cmd/api/handlers_barter.go` (`balancedBarterClearing`）
*   **ロジック**: AIに価格の加減算をさせると、ハルシネーション（計算ミス、不整合）が確実に発生します。そのため、**「計算はGoの代数公式が責任を持って行い、OpenAIはその完璧な数字をベースにした妥当性の日本語解説を作成する」** ハイブリッド設計を採用しています。
*   **数理公式**:
    $$\text{清算金（支払/受取）}_i = \text{Price}(\text{提供商品}_i) - \text{Price}(\text{受領商品}_i)$$
    これにより、参加者が何人、商品価格がいくらであっても、ループ全体の清算総和は常に $\sum \text{Adjustment} = 0$ 円（絶対的等価）に収束します。

### ③ 【完全追従】10ステップ・イベント駆動オートパイロット（自動体験ツアー）
*   **ファイル**: `frontend/src/main.tsx` (`useEffect` ディレクター)
*   **ロジック**: 時間ベースの単純なタイマー進行ではなく、**「裏側で走る非同期 Promise の解決（API成約、Stripeエスクロー決済、DALL-E生成完了）をトリガーにして次のステップへ進めるイベント駆動型」** で実装されています。
*   **マウント安全防壁**: 画面遷移時に React の仮想DOMマウントが数ミリ秒遅れるため、遷移後は 1.2秒のディレイ（同調ラグ）を挟んでからステップ信号を進めることで、DOM未描画によるイベント消失（進行デッドロック）を完全に防いでいます。

## 🎨 4. フロントエンド・コーディング規約 (Warm Minimalism)

ビジュアルの一貫性とコード品質を守るため、新機能の実装時は以下のデザインシステム規則に厳密に従ってください。

*   **基本テーマカラー (Warm Minimalism)**:
    *   背景色: `#faf8f5` (暖かい陶器のようなアイボリー)
    *   プライマリー（アクセントカラー）: `#d85b46` (朱色・マットな赤)
    *   セカンダリー（フォレスト・エスクロー）: `#275643` (深みのある緑)
*   **UIトークン**:
    *   ボタンやモーダルの角丸（Border Radius）: `30px` (または丸みのある形状)
    *   モーダルバックドロップのすりガラス効果: `backdropFilter: "blur(8px)"` と `background: "rgba(15, 23, 42, 0.65)"` を常に併用。
*   **HTMLフォーム安全ルール**:
    *   アコーディオン、タブ、決済、ARスキャナー起動、閉じるなどのモーダル制御用ボタンは、**必ず `type="button"` を明示してください**。省略するとブラウザが form 送信イベントと誤認して画面が不意に強制リロードされるバグを誘発します。
*   **画像エラーハンドリング**:
    *   外部から読み込む画像（アバター、商品写真）には、必ず `onError={(e) => { (e.target as HTMLImageElement).src = '/placeholder.svg' }}` などのフォールバック処理を記述し、 broken image を絶対に出さないようにしてください。

## 🔒 5. セキュリティ ＆ インフラ開発規約

*   **機密情報のトリミングサニタイズ**:
    GCloud Secret Manager などの環境変数からAPIキー（`OPENAI_API_KEY`）を読み込む際、末尾に改行コード（`\n`）が混入して HTTP ヘッダー不整合エラー（`invalid header field value`）を起こすことがあります。Go 側での環境変数取得時は、必ず `strings.TrimSpace(os.Getenv(...))` を通してサニタイズしてください。
*   **データベースマイグレーションの冪等性 (Idempotency)**:
    本番環境のデータベースでマイグレーションやテーブルスキーマ更新を行う際、既存のデータや作成済みのインデックスと衝突して起動パニックを起こさないよう、Go の `main.go` の `ensureIndex` 自律監査関数を介してスキーマの存在を事前に調べてからクエリを実行するようにしてください。
*   **Web MIME型 登録**:
    Cloud Run で使用される Distroless コンテナ（`gcr.io/distroless/base-debian12` など）は、オペレーティングシステムに `/etc/mime.types` ファイルを持たないため、ブラウザ側が静的アセット（JS / CSS / SVG）をセキュリティ保護（X-Content-Type-Options: nosniff）のために遮断する問題があります。
    Go 側でファイルを配信する前に、必ず `main.go` の `init()` 内部で `mime.AddExtensionType` を用いて、必要な拡張子（`.css`、`.js`、`.svg`、`.json`）の MIME タイプをプログラム内で明示的に強制登録してください。

## 🤝 6. 開発フロー ＆ コミット規約

プロジェクトへの変更をプッシュする際は、以下の規約に従ってください。

1.  **ブランチ作成**:
    ```bash
    git checkout -b feature/your-awesome-feature
    ```
2.  **検証の義務**:
    コミット前に、必ずローカルでのビルドとテストが 100% 通ることを保証してください：
    *   バックエンド: `go build ./... && go test ./...`
    *   フロントエンド: `npm run build` (tsc 型チェック＋Viteビルド)
3.  **コミットメッセージのプレフィックスルール**:
    *   `feat:` (新機能の追加)
    *   `fix:` (バグ修正、不整合の修復)
    *   `docs:` (ドキュメントやガイドの更新)
    *   `refactor:` (リファクタリング、クリーンアップ、型安全強化)
    *   `perf:` (パフォーマンス改善)
