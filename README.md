# 実装メモ

## プロダクト方針

Next Market は、OpenAI APIを使って出品者の説明文作成と購入者の質問回答を支援する次世代フリマアプリです。

## ディレクトリ

- `docs/`: UI/UX、DB、API設計
- `backend/`: Go API
- `frontend/`: React UI
- `docker-compose.yaml`: ローカルMySQL

## ローカル起動

### DB

```bash
docker compose up -d
```

### Backend

```bash
cd backend
cp .env.example .env
export $(grep -v '^#' .env | xargs)
go run ./cmd/api
```

### Frontend 開発

```bash
cd frontend
npm install
npm run dev
```

## デプロイ方針

- App: `backend/Dockerfile` で frontend をビルドして backend に同梱し、Cloud Runへデプロイ
- DB: Cloud SQL for MySQLを作成し、Cloud Runには `DATABASE_DSN` か `DB_USER` `DB_PASS` `DB_NAME` `INSTANCE_UNIX_SOCKET` を設定
- AI: Cloud Runの環境変数に `OPENAI_API_KEY` を設定

## Vercel なしで動かす

frontend を build すると backend からそのまま配信できます。

```bash
cd frontend
npm install
npm run build

cd ../backend
export $(grep -v '^#' .env | xargs)
go run ./cmd/api
```

この場合は `http://localhost:8080` を開くと、frontend と API の両方が backend から配信されます。

### Cloud Run の DB 設定メモ

Cloud Run ではローカル用の `127.0.0.1:3306` DSN をそのまま使わず、Cloud SQL 接続を設定した上で、次のどちらかで渡します。

1. `DATABASE_DSN`
2. `DB_USER`, `DB_PASS`, `DB_NAME`, `INSTANCE_UNIX_SOCKET=/cloudsql/PROJECT:REGION:INSTANCE`

`INSTANCE_UNIX_SOCKET` を使う場合は、Cloud Run サービスに Cloud SQL 接続を追加して `/cloudsql/...` が見える状態にする必要があります。

ログイン時に `database is starting` が続く場合は、Cloud Run の最小インスタンスと Cloud SQL 接続設定を確認します。手順は [docs/db-startup-troubleshooting.md](docs/db-startup-troubleshooting.md) にまとめています。

## 🏆 最新の高度な機能拡張 (2026年6月18日実装)

ハッカソンの「テーマ性・独創性」「技術的な挑戦度」「UI/UX完成度」を最高レベルに引き上げるため、以下の3大アドバンスド機能を統合しました。

### 1. 🤖 AI Agent Price Negotiation Marketplace（エージェント・フリマ）
「値下げ交渉、AIに丸投げしませんか？」をテーマに、購入者・出品者の双方が代理AIエージェントに駆け引きを完全に委託できる次世代の取引システム。
*   **自律型マルチエージェント対話**: 購入者の「希望予算」と「欲しい度」に基づき行動する購入者AIと、出品者の「最低許容価格」と「AI人格/性格」を忠実に守る出品者AIが、最大6ターンの価格交渉をバックエンド上でリアルタイムに実況チャット。
*   **エンタメ演出**: 「コテコテの大阪商人」「冷徹なエリートビジネスパーソン」「元気でかわいいアニメキャラクター」などの性格を設定可能。交渉過程がマイページや詳細モーダル上に1.5秒間隔で吹き出し出現し、ドラマチックに交渉を実況中継します。
*   **自動成約・決済**: 交渉が妥結した場合、合意した価格で即時自動購入が完了し取引が成立します。

### 2. 📊 マイ・ダッシュボード (個人分析システム)
全ユーザーがマイページからいつでも利用可能な、パーソナライズされたデータ分析画面。
*   **主要KPIカード**: 獲得売上総額、販売成立件数、出品中の商品、獲得いいね総数の4つのカードを分かりやすく集計。
*   **カテゴリー割合割合**: 自分が出品・販売した商品のカテゴリ比率を、売上額・点数のバランスも含めてグラフ化。
*   **日別売上推移グラフ**: 直近30日の自身の売上の伸び・傾向を美しいグラフタイムラインで可視化。

### 3. 🛡️ プラットフォーム管理者用ダッシュボード
プラットフォーム全体の管理・セキュリティ運用を一手に引き受けるadmin専用ダッシュボード。
*   **遷移導線**: マイページのプロフィール部に管理者権限（admin）を持つユーザーのみ「🛡️ システム管理者ダッシュボードを開く」の導線ボタンを表示。
*   **システムKPI統計**: サービス全体の総ユーザー、総出品、総取引、総売上金額の統計および推移を一元分析。
*   **AI出品審査履歴**: OpenAIの画像・テキスト解析によって検知された不適切コンテンツ・出品禁止物（リスク判定や検出されたブロックキーワード、詳細な検知理由）の履歴一覧を監視。
*   **ユーザー・ロール管理**: 登録ユーザーの一覧表示、一般ユーザー ⇄ 管理者権限のロールをドロップダウンからリアルタイムに相互変更。
